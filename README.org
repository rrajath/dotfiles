* What this is
A nix-first macOS setup. Everything that is live on this machine is declared in [[./nix-config/][nix-config/]] - a [[https://github.com/LnL7/nix-darwin][nix-darwin]] + [[https://github.com/nix-community/home-manager][home-manager]] flake with a single configuration (=default=) for user =rrajath=. There are no install scripts, no stow, no hand-symlinking: applying the flake is the only mechanism.

Apply changes with:

#+begin_src sh
sudo darwin-rebuild switch --flake ~/dotfiles/nix-config#default
#+end_src

(aliased to =drp= in nushell).

* Layout
- =nix-config/= — the flake
  - =flake.nix= — inputs and the one darwin configuration
  - =darwin.nix= — system level: nix settings, shell, TouchID sudo, Homebrew
  - =home.nix= — home-manager entry; imports every module
  - =brews.nix= / =casks.nix= — the *complete* declared Homebrew inventory
  - =modules/= — one file per app (nushell, helix, ghostty, git, jujutsu, atuin, karabiner, amethyst, claude-code, …); =packages.nix= is the nix package list
- Top-level source dirs referenced by modules: =claude-code/=, =karabiner/=, =amethyst/=
- =.emacs.d/= — literate emacs config (org files tangle to =.el=); managed *outside* home-manager. See [[./docs/EMACS.md][docs/EMACS.md]] for its current state.
- =docs/= — active backlog: [[./docs/BREW_TO_NIX.md][brew→nix migration]], [[./docs/REPO_CLEANUP.md][deferred repo cleanup]], [[./docs/EMACS.md][emacs]]
- Everything else at the top level is legacy from earlier generations of this repo, pending deletion/archival per [[./docs/REPO_CLEANUP.md][docs/REPO_CLEANUP.md]]. History: [[./CONSOLIDATION.md][CONSOLIDATION.md]].

* Conventions
- App config lives *either* inline in its nix module (default; rebuild to apply) *or* in a top-level dir wired via =mkOutOfStoreSymlink= — used when the app rewrites its own config or it churns often (karabiner, amethyst). Never both.
- Homebrew is declared but never self-updating: =onActivation.autoUpdate/upgrade = false=, =cleanup = "none"=. Activation only installs/uninstalls from the lists; run =brew update= manually.
- CLI tools prefer nixpkgs (=modules/packages.nix=); brew keeps casks, =mas=, and the genuinely brew-only.
- Version control is [[https://github.com/jj-vcs/jj][jujutsu]] (colocated with git).

* nix-config Modules
Each app gets its own module in =modules/=, imported by either =home.nix= (user-level) or =darwin.nix= (system-level). A module either declares its config inline via a Home Manager =programs.*= option (rebuild to apply) or points =xdg.configFile=/=home.file= at a top-level source dir with =mkOutOfStoreSymlink= (edits apply live, no rebuild) -  see Conventions above. What follows is every file in =modules/=: active ones wired into =home.nix=/=darwin.nix=, and inactive ones that aren't.

** Active modules
- =amethyst.nix= - out-of-store symlink of =amethyst/amethyst.yml= into =~/.config/amethyst/=; the tiling window manager's config lives outside nix so edits apply without a rebuild.
- =atuin.nix= - shell history sync with Nushell integration; up-arrow search disabled, enter-to-accept, sync of history records enabled.
- =carapace.nix= - multi-shell completion engine, Nushell integration only.
- =claude-code.nix= - symlinks =CLAUDE.md=, =.claudeignore=, and =agents/= from top-level =claude-code/= into =~/.claude= via =home.file= (not =xdg.configFile=, since Claude Code reads =$HOME= directly, not =$XDG_CONFIG_HOME=).
- =direnv.nix= - direnv with =nix-direnv= caching and Nushell integration.
- =ghostty.nix= - the active terminal: raw config text, Catppuccin Mocha theme, background blur (radius 20, full opacity), global quick-terminal toggle (=ctrl+shift+t=), =alt+left/right= unbound, tabs-style titlebar, mouse-hide-while-typing, SSH shell-integration features.
- =git.nix= - LFS enabled, ignores editor/tooling cruft (=*.swp=, =.direnv=, =.zed=, …), =core.editor = hx=, =pull.rebase= + =rebase.autoStash/updateRefs=, =rerere.enabled= (remembers conflict resolutions).
- =helix.nix= - the =$EDITOR= throughout: =base16_transparent= theme, mouse disabled, auto-save on, multi-buffer bufferline, custom cursor shapes per mode, hidden files shown in the picker, LSP messages + indent guides on, heavily remapped normal/insert keys (Emacs-style =C-a/e/n/p=, space-leader commands).
- =herdr.nix= - raw config text: catppuccin theme, =ctrl+,= key prefix, toast notifications bottom-right.
- =jjui.nix= - raw config text adding a custom =tug= action bound to =T= in the jj TUI.
- =jujutsu.nix= - installs the =jujutsu= package; configures user identity, =hx= as diff/commit editor, a =tug= bookmark-move alias, and template aliases for short IDs, relative timestamps, and local-part-only email in signatures.
- =karabiner.nix= - out-of-store symlink only; the actual remap rules live in top-level =karabiner/=, not this file.
- =macos-settings.nix= - system-level (imported by =darwin.nix=, not =home.nix=): Dock autohide/no-bounce/48px tiles, Finder column view with path/status bars, fast key repeat, trackpad tap-to-click plus three/four-finger gestures, font smoothing disabled.
- =nushell.nix= - the default shell: banner hidden, =hx= as =$EDITOR=/buffer editor, fuzzy completions, trash-on-rm, =PATH= prepends for nix/homebrew/cargo/go, and the =drp= alias for =darwin-rebuild switch=.
- =packages.nix= - not an app module; the flat Home Manager package list (dev tools, formatters, language servers, plus the brew→nix migrated tools tracked in =docs/BREW_TO_NIX.md=).
- =starship.nix= - prompt config with two custom jj-aware segments: a change-stat summary and a formatted =jj log= one-liner for the current commit, both gated on being inside a =.jj= repo.
- =vim.nix= - number/undofile/dark background via =programs.vim= (replaces the old amix/vimrc + vim-plug setup); leader-key save/quit maps and cursor-shape escape sequences in =extraConfig=.
- =zed-editor.nix= - Helix keybinding emulation enabled, system-following theme (One Light/One Dark), =nix=/=deno=/=typescript= extensions.
- =zoxide.nix= - smarter =cd=, Nushell integration only.

** Inactive modules
Present in =modules/= but not imported by =home.nix= or =darwin.nix= — leftovers from an earlier terminal/shell exploration phase, superseded by =ghostty= + =nushell=, and not applied by =darwin-rebuild=: =aerospace.nix=, =alacritty.nix=, =fish.nix=, =kitty.nix=, =wezterm.nix=, =zsh.nix=.

* Emacs
The one literate-programming survivor: =.emacs.d/PreInitConfig.org= and =PostInitConfig.org= tangle (=org-babel-tangle=) to the =.el= files emacs loads. Tangled output is gitignored by design; the org files are the source of truth.
